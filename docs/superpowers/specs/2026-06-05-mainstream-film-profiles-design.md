# Mainstream Film Profiles Design

**Date:** 2026-06-05
**Branch:** `main`
**Status:** Approved, ready for implementation planning

## Goal

Expand the film-profile set from 2 stocks (Portra 400, Fuji C200) to **13** by adding
**11 mainstream color-negative stocks**, sourced accurately from the same
`spektrafilm` dataset the existing two come from. Each profile is per-dye spectral
density data (C/M/Y `channel_density` + `base_density`) digitized from manufacturer
datasheets, flowing through the exact same `fit_m_post` → `balance_neutral` →
Mode-B inversion path the current stocks use.

## Resolved decisions

- **Source:** `spektrafilm` (Andrea Volpato), CC BY-SA 4.0 — the same dataset and
  format as the current two stocks. License posture is unchanged; the documented
  "MIT-swap before commercial release" task in `DATA_SOURCES.md` still applies to
  all bundled profile data.
- **Scope:** color negatives only (C-41 + ECN-2 Vision3). **Kodachrome 64 and all
  reversal/slide films are excluded** — they are positives (verified: Kodachrome's
  base drop ≈ 0.03, i.e. no orange mask) and would invert incorrectly in the
  negative pipeline. A non-inverting slide path is future work.
- **Conversion is reproducible and committed:** a `scripts/build_film_data.py`
  driver downloads each spektrafilm profile and writes `dye_<key>.csv`. It is the
  single source of truth and regenerates the existing two byte-for-byte (verified).

## Validation already performed

- The conversion (`np.interp` over valid samples: linear interior, edge-hold
  outside) reproduces **both** existing repo CSVs (`dye_portra400.csv`,
  `dye_fujic200.csv`) **byte-for-byte**.
- All 11 new negatives convert with physically-correct dye peaks: cyan 685–700 nm
  (red), magenta 535–545 nm (green), yellow 445–455 nm (blue), and a clear
  orange-mask base drop (0.44–1.28).

## Current state (relevant facts)

- Data lives in `crates/film-core/data/dye_<key>.csv` — columns
  `wavelength,D_C,D_M,D_Y,D_min`, grid 380–730 nm @ 5 nm (71 rows), densities
  clamped ≥ 0. Provenance in `crates/film-core/data/DATA_SOURCES.md`.
- spektrafilm JSON: `data.wavelengths` (380–780 @ 5 nm, 81 pts), `data.channel_density`
  (`[N][3]` = C/M/Y), `data.base_density` (`[N]`). `None` entries occur outside each
  curve's measured span (handled by `np.interp` edge-hold). We truncate to 380–730.
- `Stock` enum + `load_stock(stock)` (`include_str!` the CSV) live in
  `crates/film-core/src/spectral.rs`. `params_for_stock` → `fit_m_post` →
  `balance_neutral` is stock-agnostic and needs no change.
- Wire-string → `Stock` mapping is `stock_from(&str)` in `app/src-tauri/src/commands.rs`.
- Frontend `InvertParams.stock` is a string-literal union in `app/src/lib/api.ts`.
  Stock `<select>` options live in `app/src/lib/develop/Basic.svelte` and
  `app/src/lib/overlay/ConfirmDevelop.svelte`; labels use `basic.stock.<key>` i18n
  keys (`basic.noFilmProfile`, `basic.stock.portra400`, `basic.stock.fujic200`).
  i18n is generated from `i18n-strings.csv` via `scripts/gen-i18n.py`.

## Architecture / units

| Unit | Responsibility | Notes |
|---|---|---|
| `scripts/build_film_data.py` | Download spektrafilm profiles + write all `dye_*.csv` | Reproducible; idempotent for existing two |
| `crates/film-core/data/dye_<key>.csv` × 11 | Bundled per-dye density data | Generated by the script |
| `DATA_SOURCES.md` | Provenance + license + film list | Add the 11 new stocks under the same source |
| `Stock` enum + `load_stock` (`spectral.rs`) | 11 new variants + `include_str!` arms | UpperCamelCase variants (e.g. `Vision350D`) |
| `stock_from` (`commands.rs`) | wire string → `Stock` | 11 new match arms |
| `InvertParams.stock` (`api.ts`) | TS union of stock keys | add 11 literals |
| Stock `<select>` (Basic + ConfirmDevelop) | UI options | 11 new `<option>`s in each |
| i18n `basic.stock.<key>` | option labels (en + zh) | via CSV + regen |

`Stock` enum variant names (UpperCamelCase, lint-clean): `Portra160`, `Portra800`,
`Ektar100`, `Gold200`, `Ultramax400`, `FujiPro400H`, `FujiXtra400`, `Vision350D`,
`Vision3200T`, `Vision3250D`, `Vision3500T`.

## Error handling & edge cases

- **Unknown stock string:** `stock_from` returns `None` → `build_params` falls back
  to the identity (no-profile) inversion. Unchanged behavior; new stocks just add arms.
- **Old catalog blobs / persisted stock values:** any string the backend doesn't
  recognize degrades to no-profile; the TS union only constrains new writes.
- **`None`/missing spectral samples:** handled by `np.interp` edge-hold in the
  converter (already validated).
- **Tungsten Vision3 stocks (200T/500T):** valid negatives; their warm cast is
  handled by the existing auto-WB seed — no special-casing.

## Testing

- **Rust:** extend `spectral::load_stock_returns_consistent_grid` to iterate **all**
  `Stock` variants (71-point grid, equal-length curves, monotonic wavelengths). Add
  a check that every variant's `load_stock` parses and `fit_m_post` is finite/non-identity
  for each (mirrors `real_stock_yields_nontrivial_unmixing`).
- **Frontend:** `npm run check` (the `InvertParams.stock` union must include every
  `<option>` value, so a typo fails typecheck); existing unit tests stay green.
- **Data integrity:** a check (script or test) that each `dye_*.csv` has 71 data rows
  and 5 columns. The byte-for-byte reproduction of the existing two is the regression
  guard for the converter.
- **Manual:** pick several new stocks in Develop → each inverts plausibly and neutral
  (thanks to `balance_neutral`); pick one at Confirm Develop → applies to the roll.

## Out of scope (YAGNI)

- Reversal/slide (E-6, Kodachrome) support — needs a non-inverting positive path.
- `<optgroup>` grouping of the (now 13-long) stock dropdown — optional polish; keep
  flat unless it feels unwieldy in manual testing.
- Per-camera spectral sensitivity / true D55 illuminant — pre-existing model
  approximations, unchanged by this work.
- MIT relicensing of bundled data — already a separately documented pre-ship task.
